LinkedList 源码解析

1. 定义及构造方法源码

1. LinkedList 简介

  • LinkedList 实现了 List 接口 和 Deque 接口，是一个双端链表，同时也具有队列的性质。

  • LinkedList 实现了 Cloneable 接口，即覆盖了 clone() ，可以被克隆。

  • LinkedList 实现了 java.io.serializable 接口，这表示 ArrayList 支持序列化，可以通过序列化进行传输。

  • LinkedList 的链表结构使其支持高效的插入和删除操作。需要注意的是：LinkedList 虽然是链表结构，但也有索引。

  • 如果想要使 LinkedList 变成线程安全的，可以调用静态类 Collections 类中的 synchronizedList 方法：

List list = Collections.synchronizedList(new LinkedList(...));

2. 定义

LinkedList 内部定义了三个变量：size、first 和 last。

public class LinkedList<E> extends AbstractSequentialList<E> implements List<E>, Deque<E>, Cloneable, java.io.Serializable{

    transient int size = 0;

    /**
     * Pointer to first node.
     */
    transient Node<E> first;

    /**
     * Pointer to last node.
     */
    transient Node<E> last;

LinkedList 内部的链表节点有三个属性，分别是当前节点的值，前驱节点和后继节点。

private static class Node<E> {
    E item;//节点值
    Node<E> next;//后继节点
    Node<E> prev;//前驱节点

    Node(Node<E> prev, E element, Node<E> next) {
        this.item = element;
        this.next = next;
        this.prev = prev;
    }
}

3. 构造方法

LinkedList 的构造方法可以有参，也可以无参。

  1. 无参构造方法
  public LinkedList() {}
  1. 有参构造方法（用已有的集合创建链表）
    public LinkedList(Collection<? extends E> c) {
        this();
        addAll(c);
    }

可以看出，有参构造方法是调用了 addAll() 方法，后面会有介绍。

2. 添加方法源码

1. linkLast / linkFirst 方法

  • 最核心的添加方法 linkLast(E e) ：
   /**
     * 链接使e作为最后一个元素。
     * 其中last是链表中指向最后一个节点的指针
     * 其中first是链表中指向第一个节点的指针
     * l的作用是记录上一个节点
     */
    void linkLast(E e) {
        final Node<E> l = last;
        final Node<E> newNode = new Node<>(l, e, null);
        last = newNode;//新建节点
        if (l == null)
            first = newNode;
        else
            l.next = newNode;//指向后继元素也就是指向下一个元素
        size++;
        modCount++;
    }
  1. 先新建引用（l）记录上一个节点，也就是之前的最后一个节点（last）；
  2. 新建节点 newNode，前驱指向 l，后继为 null；
  3. 令 last 指向新建节点，此时新建节点是链表的最后一个节点；
  4. 判断上一节点 l 是否为 null，是则说明新节点是第一个节点，first 也要指向新节点；不是则将上一节点的后继指向新节点。
  • 从头部添加的方法linkFirst(E e) ：
    /**
      * 操作方法步骤与上述linkLast方法相同
      */
    private void linkFirst(E e) {
        final Node<E> f = first;
        final Node<E> newNode = new Node<>(null, e, f);//新建节点，以头节点为后继节点
        first = newNode;
        //如果链表为空，last节点也指向该节点
        if (f == null)
            last = newNode;
        //否则，将头节点的前驱指针指向新节点，也就是指向前一个元素
        else
            f.prev = newNode;
        size++;
        modCount++;
    }

2. add 方法

  • add(E e) 方法：将元素添加到链表尾部。
    public boolean add(E e) {
        linkLast(e);//这里就只调用了这一个方法
        return true;
    }

可见，add() 就是调用了 linkLast() 方法，返回 true 则说明添加成功。

  • add(int index, E e) ：将元素添加到指定位置。
    public void add(int index, E element) {
        checkPositionIndex(index); //检查索引是否处于[0-size]之间

        if (index == size)//添加在链表尾部
            linkLast(element);
        else//添加在链表中间
            linkBefore(element, node(index));
    }

linkBefore() 方法又需其他条件。只需记住，LinkedList 的插入添加操作比 ArrayList 效率更高。

  • addFirst(E e) ：将元素添加到链表头部。
     public void addFirst(E e) {
        linkFirst(e);
    }
  • addLast(E e) ：将元素添加到链表尾部。
    public void addLast(E e) {
        linkLast(e);
    }

向尾部添加元素一般使用 add() 方法。

3. addAll 方法

  • addAll(Collection c) ：将集合添加到链表尾部。
    public boolean addAll(Collection<? extends E> c) {
        return addAll(size, c);
    }

调用的是重载方法，也就是从链表的最后开始插入。

  • addAll(int index, Collection c) ：将集合添加到链表指定位置。
    public boolean addAll(int index, Collection<? extends E> c) {
        //1:检查index范围是否在size之内
        checkPositionIndex(index);

        //2:toArray()方法把集合的数据存到对象数组中
        Object[] a = c.toArray();
        int numNew = a.length;
        if (numNew == 0)
            return false;

        //3：得到插入位置的前驱节点和后继节点
        Node<E> pred, succ;
        //如果插入位置为尾部，前驱节点为last，后继节点为null
        if (index == size) {
            succ = null;
            pred = last;
        }
        //否则，调用node()方法得到后继节点，再得到前驱节点
        else {
            succ = node(index);
            pred = succ.prev;
        }

        // 4：遍历数据将数据插入
        for (Object o : a) {
            @SuppressWarnings("unchecked") E e = (E) o;
            //创建新节点
            Node<E> newNode = new Node<>(pred, e, null);
            //如果插入位置在链表头部
            if (pred == null)
                first = newNode;
            else
                pred.next = newNode;
            pred = newNode;
        }

        //如果插入位置在尾部，重置last节点
        if (succ == null) {
            last = pred;
        }
        //否则，将插入的链表与先前链表连接起来
        else {
            pred.next = succ;
            succ.prev = pred;
        }

        size += numNew;
        modCount++;
        return true;
    }
  1. 检查index范围是否在 size 之内；
  2. 调用 toArray() 方法把集合的数据存到对象数组中（Object 数组）；
  3. 调用 node() 方法得到插入位置的前驱和后继节点；
  4. 遍历数据，将数据插入到指定位置。

3. 根据索引获取对象的方法源码

1. node 方法

  • 最核心的根据索引获取数据的方法 node(int index) ：
    /**
     * 返回一个指定索引的非空节点.
     */
    Node<E> node(int index) {
        // assert isElementIndex(index);

        if (index < (size >> 1)) {
            Node<E> x = first;
            for (int i = 0; i < index; i++)
                x = x.next;
            return x;
        } else {
            Node<E> x = last;
            for (int i = size - 1; i > index; i--)
                x = x.prev;
            return x;
        }
    }

node() 方法的机制是：先判断指定位置是在链表的前半段还是后半段，然后从较近的一端开始遍历，直到找到指定元素。

2. get 方法

  • get(int index) ：根据指定索引获取数据。
    public E get(int index) {
        //检查index范围是否在size之内
        checkElementIndex(index);
        //调用Node(index)去找到index对应的node然后返回它的值
        return node(index).item;
    }

可见，get() 方法是调用了 node() 方法实现的。

3. 获取头节点的方法

  • 获取头节点的方法：
    public E getFirst() {
        final Node<E> f = first;
        if (f == null)
            throw new NoSuchElementException();
        return f.item;
    }

    //element()就是getFirst()实现的，二者没有区别
    public E element() {
        return getFirst();
    }

    public E peek() {
        final Node<E> f = first;
        return (f == null) ? null : f.item;
    }
.
    //与peek()没有区别
    public E peekFirst() {
        final Node<E> f = first;
        return (f == null) ? null : f.item;
     }

使用和区别：常用方法为 getFirst() 和 peek()/peekFirst() 。二者的区别在于，当链表为空时（头节点为空则链表为空），getFirst() 方法会抛出异常，peek() 方法会返回 null。

4. 获取尾节点的方法

  • 获取尾节点的方法：
     public E getLast() {
        final Node<E> l = last;
        if (l == null)
            throw new NoSuchElementException();
        return l.item;
    }

     public E peekLast() {
        final Node<E> l = last;
        return (l == null) ? null : l.item;
    }

区别：当链表为空时（尾节点为空则链表为空），getLast() 方法会抛出异常，peekLast() 方法会返回 null。

4. 根据对象获得索引的方法源码

1. indexOf 方法

  • int indexOf(Object o) ：从头遍历寻找。
    public int indexOf(Object o) {
        int index = 0;
        if (o == null) {
            //从头遍历
            for (Node<E> x = first; x != null; x = x.next) {
                if (x.item == null)
                    return index;
                index++;
            }
        } else {
            //从头遍历
            for (Node<E> x = first; x != null; x = x.next) {
                if (o.equals(x.item))
                    return index;
                index++;
            }
        }
        return -1;
    }

2. lastIndexOf 方法

  • int lastIndexOf(Object o) ：从尾遍历寻找。
    public int lastIndexOf(Object o) {
        int index = size;
        if (o == null) {
            //从尾遍历
            for (Node<E> x = last; x != null; x = x.prev) {
                index--;
                if (x.item == null)
                    return index;
            }
        } else {
            //从尾遍历
            for (Node<E> x = last; x != null; x = x.prev) {
                index--;
                if (o.equals(x.item))
                    return index;
            }
        }
        return -1;
    }

3. contains 方法

  • contains(Object o) ：检查对象 o 是否存在于链表中。
     public boolean contains(Object o) {
        return indexOf(o) != -1;
    }

可见，contains() 方法就是调用了 indexOf() 方法，判断返回值是否合理。

5. 删除方法源码

1. unlink 方法

  • 最核心的删除方法 unlink(Node x) ：
    E unlink(Node<E> x) {
        // assert x != null;
        final E element = x.item;
        final Node<E> next = x.next;//得到后继节点
        final Node<E> prev = x.prev;//得到前驱节点

        //删除前驱指针
        if (prev == null) {
            first = next;//如果删除的节点是头节点,令头节点指向该节点的后继节点
        } else {
            prev.next = next;//将前驱节点的后继节点指向后继节点
            x.prev = null;
        }

        //删除后继指针
        if (next == null) {
            last = prev;//如果删除的节点是尾节点,令尾节点指向该节点的前驱节点
        } else {
            next.prev = prev;
            x.next = null;
        }

        x.item = null;
        size--;
        modCount++;
        return element;
    }

unlink() 方法的作用就是断开指定节点的前驱和后继，是删除方法中的核心。

2. remove 方法

remove方法共有三种重载形式，每种形式的作用都不相同。

  1. remove() ：空参形式，删除头节点，后面介绍。
  2. remove(Object o) ：删除指定元素。
    public boolean remove(Object o) {
        //如果删除对象为null
        if (o == null) {
            //从头开始遍历
            for (Node<E> x = first; x != null; x = x.next) {
                //找到元素
                if (x.item == null) {
                   //从链表中移除找到的元素
                    unlink(x);
                    return true;
                }
            }
        } else {
            //从头开始遍历
            for (Node<E> x = first; x != null; x = x.next) {
                //找到元素
                if (o.equals(x.item)) {
                    //从链表中移除找到的元素
                    unlink(x);
                    return true;
                }
            }
        }
        return false;
    }
  • 需要先遍历链表找到此元素，再调用 unlink() 方法删除。遍历方式为从头遍历。
  • 此方法一次只会删除一个匹配的对象，成功返回 true。
  1. remove(int index) ：删除指定位置的元素。
    public E remove(int index) {
        //检查index范围
        checkElementIndex(index);
        //将节点删除
        return unlink(node(index));
    }
  • 先调用 node() 方法找到该元素，再调用 unlink() 方法删除。

3. 删除头节点的方法

    public E pop() {
        return removeFirst();
    }

    public E remove() {
        return removeFirst();
    }

    public E removeFirst() {
        final Node<E> f = first;
        if (f == null)
            throw new NoSuchElementException();
        return unlinkFirst(f);
    }
  • 三者实为同一种方法，其他方法都是调用 removeFirst() ，当链表为空时，会抛出异常。使用时也尽量使用此方法。

4. 删除尾节点的方法

    public E removeLast() {
        final Node<E> l = last;
        if (l == null)
            throw new NoSuchElementException();
        return unlinkLast(l);
    }

    public E pollLast() {
        final Node<E> l = last;
        return (l == null) ? null : unlinkLast(l);
    }
  • 区别：当链表为空时，removeLast() 方法会抛出异常，而 pollLast() 方法会返回 null。

6. LinkedList 面试问题总结

1. ArrayList 和 LinkedList 的对比

  1. 线程安全：ArrayList 和 LinkedList 都是线程不同步的，不保证线程安全。
  2. 底层数据结构：ArrayList 的底层数据结构是 Object 数组；LinkedList 底层结构是双向链表。
  3. 插入和删除：ArrayList 插入和删除元素受元素位置的影响，时间复杂度较大；LinkedList 插入删除元素不受元素位置的影响，时间复杂度较小，但首先要移动到指定位置，这个过程时间复杂度较大。
  4. 快速随机访问（查找元素）：ArrayList 可以通过序号快速获取元素对象；LinkedList 不能，但也做了优化，先用二分法判断靠近哪一端，再从较近的一端开始查找。
  5. 内存空间占用：ArrayList 空间浪费体现在预留一定的容量空间；LinkedList 空间花费体现在，它的每一个元素都要多储存前驱和后继。
